# Comprehensive Macro Test Plan (StateKit)

This document outlines the testing strategy for the 48+ Swift Macros implemented in StateKit to ensure reliability, correctness, and a "real-world ready" developer experience.

## Strategy Tiers

### Tier 1: Macro Expansion & Diagnostic Testing (Unit TIER)
**Goal**: Verify that macros generate the exact Swift code intended and provide helpful error messages for misuse.

- **Tools**: `SwiftSyntaxMacrosTestSupport`, `assertMacroExpansion`.
- **Status**: **100% Covered (52 Test Cases)**.
- **Key Files**:
    - `AtomMacrosTests.swift`: 17 tests (Core & Derived Atoms).
    - `ViewMacrosTests.swift`: 5 tests (UI & Observation).
    - `HookMacrosTests.swift`: 16 tests (Hooks & Side Effects).
    - `RiverpodMacrosTests.swift`: 11 tests (Providers & Families).
    - `MacroTests.swift`: Infrastructure checks.

### Tier 2: Integration & Runtime Testing (Integration TIER)
**Goal**: Ensure the code generated by macros compiles against the full library and behaves correctly at runtime.

- **Strategy**: Create a specialized test target `StateKitMacroIntegrationTests` that uses the macros in real code and asserts their runtime behavior.
- **Verification points**:
    - Do `@StateAtom` extensions actually satisfy protocol requirements?
    - Does `@Provider` generate a property that can be accessed via `ProviderContainer`?
    - Does `@HookView` actually wrap the body in a `StateScope` at runtime?

### Tier 3: Edge Case & Robustness Validation
**Goal**: Verify macros handle non-standard Swift syntax and complex scenarios.

- **Checklist**:
    - **Access Levels**: Test with `public`, `internal`, `private`, and `fileprivate` declarations.
    - **Generic Types**: Test macros with generic structs/functions (e.g., `@ValueAtom struct MyAtom<T>`).
    - **Type Aliases**: Test if macros handle typealiases correctly in method signatures.
    - **Nested Types**: Test macros applied to types nested inside other types.
    - **Whitespace/Formatting**: Verify macros don't break on unconventional formatting.

### Tier 4: Tooling & IDE Compatibility
**Goal**: Ensure macros play well with Xcode/SourceKit (Autocomplete, Refactoring).

- **Verification**: Manual audit in Xcode to ensure generated members (`Value`, `Provider`, `useHook`) appear in autocomplete and don't cause red herrings in the editor.
